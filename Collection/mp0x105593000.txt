      <div class="article-body">
        <p>Antes de começarmos: se você tiver qualquer problema durante o beta aberto, o primeiro passo é sempre reiniciar o cliente. Isso deve resolver a maioria dos problemas. E lembre-se, se você acabar tendo muitos problemas, sinta-se livre para voltar a usar o cliente legado.</p>

<h2>Meu problema não está nessa lista!</h2>
<p>Se você encontrar um problema que não está listado aqui, use o botão de Bugs, localizado na parte inferior da sua lista de amigos. Esses relatórios são enviados para a equipe responsável pelo cliente atualizado e nos ajudam a monitorar e eliminar bugs mais rapidamente. Agradecemos a sua ajuda!</p>
<p><a name="h1q1"></a></p>
<h2>Como faço para voltar a usar o cliente legado?</h2>
<p>Basta localizar o texto laranja “Iniciar o cliente legado”, localizado na parte inferior direita da tela de início de sessão.</p>
<p><a name="h1q2"></a></p>
<h2>Como envio relatórios do cliente atualizado durante o beta aberto?</h2>
<p>Enquanto estamos ajudando você a resolver alguns problemas do cliente atualizado ou buscando melhorar o futuro do cliente, há momentos em que seus relatórios podem ser muito úteis para nós, pois vão nos contar o que está acontecendo.</p>
<p>A informação é sempre muito útil, mas tenha em mente que o cliente atualizado ainda está em fase beta, então mesmo enviando relatórios, pode ser que ainda não tenhamos uma maneira de resolver o seu problema.</p>
<blockquote>Usuários de Mac: Clique <a href="https://support.riotgames.com/hc/en-us/articles/201752674#mq11">aqui</a> para saber como obter os relatórios.</blockquote>
<ul>
<ul>
<li>Como encontro meus relatórios?</li>
</ul>
</ul>
<p>Acesse sua pasta do League of Legends (costuma estar localizada em C:\Riot Games\League of Legends\) e abra a pasta “Logs”. Nessa pasta você encontrará outras pastas, com arquivos dentro.</p>
<ul>
<ul>
<li>Beleza, agora eu já sei onde estão os relatórios. O que eu faço com eles?</li>
</ul>
</ul>
<p>Você vai escolher um programa de compressão de arquivos (como WinZip, 7Zip, etc) e compactar todas essas pastas em um único arquivo .zip. Quando os arquivos estiverem compactados, você vai <a href="https://support.riotgames.com/hc/en-us/requests/new">criar um ticket</a> alegando problemas técnicos e anexar esse arquivo lá. No corpo do ticket, não esqueça de especificar que esse é um problema relacionado ao cliente atualizado.</p>
<p><img src="/hc/en-us/article_attachments/217283507/TechTicket.png" alt="" width="1000" /></p>
<blockquote><strong>Obs.:</strong> Para enviar um ticket, é necessário entrar com a sua conta!</blockquote>
<p><a name="h1q3"></a></p>
<h2>Reinstalei o cliente atualizado mas continuo tendo erros de BugSplat.</h2>
<p>Se você desinstalar ou reinstalar o cliente atualizado, pode notar erros de BugSplat mais frequentes. O cliente atualizado compartilha arquivos com o cliente legado, então, ao desinstalar um, você afeta o outro.</p>
<p>Para evitar erros de BugSplat, desinstale todos os arquivos de League of Legends do seu computador e faça uma instalação limpa.</p>
<p><a name="h1q4"></a></p>
<h2>Meu cliente estava atualizando, chegou a 99% e depois a porcentagem caiu</h2>
<p>Isso é apenas um erro visual. Deixe a atualização continuar e logo o 100% chega!</p>
<p><a name="h1q5"></a></p>
<h2>Minha atualização está travada em 87%</h2>
<p>Isso é um erro visual (você provavelmente já ficou com a atualização travada em 33%), o jogo continuará sendo atualizado normalmente. Não interrompa o processo, a atualização será concluída.</p>
<p><a name="h1q6"></a></p>
<h2>Uso o Windows XP e, ao atualizar meu cliente, vejo uma janela acusando o erro “RunDLL”.</h2>
<p>Usuários do Windows XP receberão uma mensagem de erro durante o processo de atualização do cliente. Isso não impede o processo de ser concluído com sucesso. Se você usa o XP e recebeu essa mensagem, basta fechá-la e curtir o cliente atualizado. É importante ter em mente que, ao usar o cliente atualizado no Windows XP, é esperado que você veja essa mensagem 3 vezes.</p>
<p><a name="h1q7"></a></p>
<h2>Não consigo encontrar uma opção para aderir ao Beta no Atualizador.</h2>
<p>É possível que isso aconteça porque você tem um ícone do Atualizador antigo fixado na sua barra de tarefas. Para resolver isso, basta remover o atalho do League of Legends da sua barra de tarefas, acessar <strong>C:\Riot Games\League of Legends</strong> e clicar em <strong>lol.launcher</strong>. Isso vai atualizar o seu Atualizador e fazer com que você veja o pop-up (ou artigo) quando a atualização terminar. O pop-up do Atualizador é como o mostrado na primeira imagem. Já o artigo, você confere na segunda.</p>
<p><img src="/hc/en-us/article_attachments/115001988108/Client-Popup.jpg" /> <img src="/hc/en-us/article_attachments/115001965187/Client-Article.jpg" /></p>
<p><a name="h1q8"></a></p>
<h2>Recebi o seguinte erro: “LCU encontrou um problema. Reinstale o cliente alfa de League para tentar novamente”.</h2>
<p>Para resolver isso, siga os passos abaixo:</p>
<ul>
<li>Acesse C:\Riot Games\League of Legends\ e exclua o arquivo LeagueClient.exe</li>
<li>Execute o arquivo lol.launcher.exe</li>
<li>Abra as configurações e clique em Reparo Completo</li>
</ul>
<p><a name="h2q1"></a></p>
<h2>Minha Honra não aparece no meu perfil e na tela de Seleção de Campeões</h2>
<p>Sabemos de um problema que pode impedir as insígnias de honra de aparecerem no seu perfil e durante a Seleção de Campeões. Ainda estamos contabilizando os pontos de Honra, mas sua insígnia deve ser exibida na tela de carregamento e tela de fim de partida.</p>
<p><a name="h2q2"></a></p>
<h2>O cliente atualizado não é reconhecido quando faço streams</h2>
<p>O cliente atualizado é renderizado de maneira diferente do cliente legado na maioria das placas de vídeo, então, dependendo das suas configurações, pode ser que ele não seja exibido no seu software de transmissão. Se o seu software possui configurações com base em Captura de Jogo, mude para Captura de Janela ou Captura de Área de Trabalho, dependendo do que funciona melhor para você. Não deixe de verificar se você está usando a versão correta do software de transmissão (32 ou 64-bit), já que problemas de compatibilidade também podem impedir que o cliente seja capturado corretamente.</p>
<p>Se você preferir não alterar suas configurações, a partida será capturada corretamente quando começar. Esse problema só afeta o cliente atualizado.</p>
<p><a name="h2q3"></a></p>
<p>Não consigo selecionar os modos de jogo PvP ou Co-op vs. IA</p>
<p>Basta reiniciar o seu cliente.</p>
<p><a name="h2q4"></a></p>
<h2>Não consigo selecionar a dificuldade Intermediário ao jogar Co-op vs. IA em Twisted Treeline</h2>
<p>Estamos cientes desse problema e planejamos resolvê-lo o quanto antes!</p>
<p><a name="h2q5"></a></p>
<h2>O bônus de XP/IP que comprei não está aparecendo na minha conta</h2>
<p>Isso é apenas um bug visual. Basta reiniciar seu cliente e ele será exibido normalmente!</p>
<p><a name="h2q6"></a></p>
<h2>Alterar arquivos de configuração para jogar em outro idioma que não seja suportado pela sua região irá criar um “erro não específico” que não vai parar de ocorrer.</h2>
<p>Se você modificou manualmente os arquivos de configuração para jogar em um idioma não suportado pela sua região, você receberá um "erro não específico" quando optar por usar o cliente beta e tentar fazer o download do cliente atualizado. Para consertar isso, acesse a pasta onde o League está instalado e percorra o seguinte caminho: /RADS/system/locale.cfg. Feito isso, determine "locale" para o código local de um dos idiomas que a sua região suporta (para o Brasil, pt_br).</p>
<p><a name="h3q1"></a></p>
<h2>Eu fecho o cliente do League mas o Leagueclient.exe continua sendo executado e leva um tempo até ser encerrado. Se eu tento abrir o jogo novamente, a mensagem “Há outra instância em execução” aparece.</h2>
<p>Estamos cientes do problema que impede o LeagueClient.exe de ser encerrado corretamente quando você sai do jogo. Caso isso aconteça com você, pressione CTRL+ALT+DELETE para acessar o Gerenciador de Tarefas e feche manualmente o processo do LeagueClient.exe.</p>
<p><a name="h3q2"></a></p>
<h2>Meu computador não é tão potente, o cliente atualizado fica superlento!</h2>
<p>Nossa principal prioridade é melhorar o desempenho. Identificamos desempenhos bem ruins em máquinas menos potentes que tentam rodar o cliente atualizado. Continuaremos melhorando o cliente para garantir que jogadores que utilizam o cliente legado sem problemas possam usar o cliente atualizado da mesma maneira.</p>
<p>Existem algumas coisas que você pode fazer para melhorar o desempenho, no entanto:</p>
<ul>
<li>Certifique-se de que você está usando drivers atualizados para a sua placa de vídeo</li>
<li>Ative o modo "Configuração Baixa". Você pode ativá-lo através das configurações do cliente, clicando na engrenagem localizada no canto superior direito da sua tela</li>
</ul>

<h2>Meu problema não está nessa lista!</h2>
<p>Se você encontrar um problema que não está listado aqui, use o botão de Bugs, localizado na parte inferior da sua lista de amigos. Esses relatórios são enviados para a equipe responsável pelo cliente atualizado e nos ajudam a monitorar e eliminar bugs mais rapidamente. Agradecemos a sua ajuda!</p>
<p><a name="h1q1"></a></p>
<h2>Como faço para voltar a usar o cliente legado?</h2>
<p>Basta localizar o texto laranja “Iniciar o cliente legado”, localizado na parte inferior direita da tela de início de sessão.</p>
<p><a name="h1q2"></a></p>
<h2>Como envio relatórios do cliente atualizado durante o beta aberto?</h2>
<p>Enquanto estamos ajudando você a resolver alguns problemas do cliente atualizado ou buscando melhorar o futuro do cliente, há momentos em que seus relatórios podem ser muito úteis para nós, pois vão nos contar o que está acontecendo.</p>
<p>A informação é sempre muito útil, mas tenha em mente que o cliente atualizado ainda está em fase beta, então mesmo enviando relatórios, pode ser que ainda não tenhamos uma maneira de resolver o seu problema.</p>
<blockquote>Usuários de Mac: Clique <a href="https://support.riotgames.com/hc/en-us/articles/201752674#mq11">aqui</a> para saber como obter os relatórios.</blockquote>
<ul>
<ul>
<li>Como encontro meus relatórios?</li>
</ul>
</ul>
<p>Acesse sua pasta do League of Legends (costuma estar localizada em C:\Riot Games\League of Legends\) e abra a pasta “Logs”. Nessa pasta você encontrará outras pastas, com arquivos dentro.</p>
<ul>
<ul>
<li>Beleza, agora eu já sei onde estão os relatórios. O que eu faço com eles?</li>
</ul>
</ul>
<p>Você vai escolher um programa de compressão de arquivos (como WinZip, 7Zip, etc) e compactar todas essas pastas em um único arquivo .zip. Quando os arquivos estiverem compactados, você vai <a href="https://support.riotgames.com/hc/en-us/requests/new">criar um ticket</a> alegando problemas técnicos e anexar esse arquivo lá. No corpo do ticket, não esqueça de especificar que esse é um problema relacionado ao cliente atualizado.</p>
<p><img src="/hc/en-us/article_attachments/217283507/TechTicket.png" alt="" width="1000" /></p>
<blockquote><strong>Obs.:</strong> Para enviar um ticket, é necessário entrar com a sua conta!</blockquote>
<p><a name="h1q3"></a></p>
<h2>Reinstalei o cliente atualizado mas continuo tendo erros de BugSplat.</h2>
<p>Se você desinstalar ou reinstalar o cliente atualizado, pode notar erros de BugSplat mais frequentes. O cliente atualizado compartilha arquivos com o cliente legado, então, ao desinstalar um, você afeta o outro.</p>
<p>Para evitar erros de BugSplat, desinstale todos os arquivos de League of Legends do seu computador e faça uma instalação limpa.</p>
<p><a name="h1q4"></a></p>
<h2>Meu cliente estava atualizando, chegou a 99% e depois a porcentagem caiu</h2>
<p>Isso é apenas um erro visual. Deixe a atualização continuar e logo o 100% chega!</p>
<p><a name="h1q5"></a></p>
<h2>Minha atualização está travada em 87%</h2>
<p>Isso é um erro visual (você provavelmente já ficou com a atualização travada em 33%), o jogo continuará sendo atualizado normalmente. Não interrompa o processo, a atualização será concluída.</p>
<p><a name="h1q6"></a></p>
<h2>Uso o Windows XP e, ao atualizar meu cliente, vejo uma janela acusando o erro “RunDLL”.</h2>
<p>Usuários do Windows XP receberão uma mensagem de erro durante o processo de atualização do cliente. Isso não impede o processo de ser concluído com sucesso. Se você usa o XP e recebeu essa mensagem, basta fechá-la e curtir o cliente atualizado. É importante ter em mente que, ao usar o cliente atualizado no Windows XP, é esperado que você veja essa mensagem 3 vezes.</p>
<p><a name="h1q7"></a></p>
<h2>Não consigo encontrar uma opção para aderir ao Beta no Atualizador.</h2>
<p>É possível que isso aconteça porque você tem um ícone do Atualizador antigo fixado na sua barra de tarefas. Para resolver isso, basta remover o atalho do League of Legends da sua barra de tarefas, acessar <strong>C:\Riot Games\League of Legends</strong> e clicar em <strong>lol.launcher</strong>. Isso vai atualizar o seu Atualizador e fazer com que você veja o pop-up (ou artigo) quando a atualização terminar. O pop-up do Atualizador é como o mostrado na primeira imagem. Já o artigo, você confere na segunda.</p>
<p><img src="/hc/en-us/article_attachments/115001988108/Client-Popup.jpg" /> <img src="/hc/en-us/article_attachments/115001965187/Client-Article.jpg" /></p>
<p><a name="h1q8"></a></p>
<h2>Recebi o seguinte erro: “LCU encontrou um problema. Reinstale o cliente alfa de League para tentar novamente”.</h2>
<p>Para resolver isso, siga os passos abaixo:</p>
<ul>
<li>Acesse C:\Riot Games\League of Legends\ e exclua o arquivo LeagueClient.exe</li>
<li>Execute o arquivo lol.launcher.exe</li>
<li>Abra as configurações e clique em Reparo Completo</li>
</ul>
<p><a name="h2q1"></a></p>
<h2>Minha Honra não aparece no meu perfil e na tela de Seleção de Campeões</h2>
<p>Sabemos de um problema que pode impedir as insígnias de honra de aparecerem no seu perfil e durante a Seleção de Campeões. Ainda estamos contabilizando os pontos de Honra, mas sua insígnia deve ser exibida na tela de carregamento e tela de fim de partida.</p>
<p><a name="h2q2"></a></p>
<h2>O cliente atualizado não é reconhecido quando faço streams</h2>
<p>O cliente atualizado é renderizado de maneira diferente do cliente legado na maioria das placas de vídeo, então, dependendo das suas configurações, pode ser que ele não seja exibido no seu software de transmissão. Se o seu software possui configurações com base em Captura de Jogo, mude para Captura de Janela ou Captura de Área de Trabalho, dependendo do que funciona melhor para você. Não deixe de verificar se você está usando a versão correta do software de transmissão (32 ou 64-bit), já que problemas de compatibilidade também podem impedir que o cliente seja capturado corretamente.</p>
<p>Se você preferir não alterar suas configurações, a partida será capturada corretamente quando começar. Esse problema só afeta o cliente atualizado.</p>
<p><a name="h2q3"></a></p>
<p>Não consigo selecionar os modos de jogo PvP ou Co-op vs. IA</p>
<p>Basta reiniciar o seu cliente.</p>
<p><a name="h2q4"></a></p>
<h2>Não consigo selecionar a dificuldade Intermediário ao jogar Co-op vs. IA em Twisted Treeline</h2>
<p>Estamos cientes desse problema e planejamos resolvê-lo o quanto antes!</p>
<p><a name="h2q5"></a></p>
<h2>O bônus de XP/IP que comprei não está aparecendo na minha conta</h2>
<p>Isso é apenas um bug visual. Basta reiniciar seu cliente e ele será exibido normalmente!</p>
<p><a name="h2q6"></a></p>
<h2>Alterar arquivos de configuração para jogar em outro idioma que não seja suportado pela sua região irá criar um “erro não específico” que não vai parar de ocorrer.</h2>
<p>Se você modificou manualmente os arquivos de configuração para jogar em um idioma não suportado pela sua região, você receberá um "erro não específico" quando optar por usar o cliente beta e tentar fazer o download do cliente atualizado. Para consertar isso, acesse a pasta onde o League está instalado e percorra o seguinte caminho: /RADS/system/locale.cfg. Feito isso, determine "locale" para o código local de um dos idiomas que a sua região suporta (para o Brasil, pt_br).</p>
<p><a name="h3q1"></a></p>
<h2>Eu fecho o cliente do League mas o Leagueclient.exe continua sendo executado e leva um tempo até ser encerrado. Se eu tento abrir o jogo novamente, a mensagem “Há outra instância em execução” aparece.</h2>
<p>Estamos cientes do problema que impede o LeagueClient.exe de ser encerrado corretamente quando você sai do jogo. Caso isso aconteça com você, pressione CTRL+ALT+DELETE para acessar o Gerenciador de Tarefas e feche manualmente o processo do LeagueClient.exe.</p>
<p><a name="h3q2"></a></p>
<h2>Meu computador não é tão potente, o cliente atualizado fica superlento!</h2>
<p>Nossa principal prioridade é melhorar o desempenho. Identificamos desempenhos bem ruins em máquinas menos potentes que tentam rodar o cliente atualizado. Continuaremos melhorando o cliente para garantir que jogadores que utilizam o cliente legado sem problemas possam usar o cliente atualizado da mesma maneira.</p>
<p>Existem algumas coisas que você pode fazer para melhorar o desempenho, no entanto:</p>
<ul>
<li>Certifique-se de que você está usando drivers atualizados para a sua placa de vídeo</li>
<li>Ative o modo "Configuração Baixa". Você pode ativá-lo através das configurações do cliente, clicando na engrenagem localizada no canto superior direito da sua tela</li>
</ul>

            var cpuGood = true; //true by default because we don't want to give warnings when we're not sure or can't parse the CPU speed
            if (cpuName.toLowerCase().indexOf('ghz') != -1) {
                //GHZ info is available to compare against
                var cpuSpeed = parseFloat(cpuName.substr(cpuName.toLowerCase().indexOf('ghz') - 4, 4));
                if (cpuSpeed) {
                    cpuGood = cpuSpeed >= 2.0;
                    $('.kbCantCheckCpuSpeed').hide();
                } else {
                    $('.kbCantCheckCpuSpeed').show();
                }
            } else {
                $('.kbCantCheckCpuSpeed').show();
            }
			if (crashed || fpsUhOh)
			{
				if (gpuRamRecommended) {
                    $('.kbHardwareBelowRecSpec').hide();
                }
                if (ramGood && cpuGood) {
                    $('.kbHardwareBad').hide();
                } else if (ramGood) {
                    $('.kbHardwareGood').hide();
                    $('.kbBelowRamMin').hide();
                    $('.kbRamAndCPU').hide();
                } else if (cpuGood) {
                    $('.kbHardwareGood').hide();
                    $('.kbCpuBelowMin').hide();
                    $('.kbRamAndCPU').hide();
                } else {
                    $('.kbHardwareGood').hide();
                }
			}
            if (crashed) {
                $('.kbIssueDetected').show();
                $('.kbFPS').hide();
				$('.kbFpsIntro').hide();
				$('.kbFPSGeneral').hide();
				
            } else if (fpsUhOh) {
                $('.kbIssueDetected').show();
                $('.kbCrashing').hide();
                $('.kbCrashingIntro').hide();
                $('.kbFpsAvg').html(fpsAvg);
                $('.kbFpsMin').html(fpsMin);
				$('.kbCrashGeneral').hide();
            }
			if (firewallIssues) {
				$('.kbFirewall').show();
			}
			if (!crashed && !fpsUhOh && !firewallIssues) {
				$('.kbAllClear').show();
			}
			$('html, body').animate({scrollTop: $('h2.hardwareInfoHeader').first().offset().top}, 2000);
        }
        r.readAsText(f);
    } else
        alert("Failed to open file");

}
function readSingleFile(evt) {
  var contentsArray;
	$('#majorissues').hide();
	$('#somepingissues').hide();
	$('#somepacketlossissues').hide();
	$('#noissues').hide();
	$('#basicsolutions').hide();
    var packetLossArray = new Array();
	var packetSentArray = new Array();
    var pingArray = new Array();
	var packetLossPctArray = new Array();
	var variance = 0;
    var f = evt.target.files[0];
    if (f) {
        var r = new FileReader();
        r.onload = function(e) {
            var contents = e.target.result;
			if (contents.indexOf('X.X.X.X')==-1)
			{
				alert ("This doesn't seem to be a netlog; please try again!");
				return;
			}
            contents = contents.substring(contents.indexOf('X.X.X.X') + 8);
			var date = new Date();
            contents = contents.substring(0, contents.indexOf('Memory Allocation')-21);
            contentsArray = contents.split('\n');
            contentsArray.pop();
            var i = 1, lastLoss=0, lastPct=0,lastPing=0;
			var notLogLine = false;
			var breakNow = false;
            for (i = 1; i < contentsArray.length; i++) {
				if (notLogLine) i--;
				notLogLine = false;
                var myLogLine = contentsArray[i].split(',');
				while (myLogLine.length < 9){
					i++;
					notLogLine = true;
					if (contentsArray[i])
						myLogLine = contentsArray[i].split(',');
					else{
						breakNow=true;
						break;
					}
				}
				if (breakNow)break;
				
				var losss= (myLogLine[6] / myLogLine[7])*100
				if (losss==lastPct || myLogLine[6] == lastLoss){
					packetLossPctArray.push(0);
					packetLossArray.push(0); 
					packetSentArray.push(myLogLine[7]);
				}
				else{
					packetLossPctArray.push(losss);
					packetLossArray.push(myLogLine[6]); 
					packetSentArray.push(myLogLine[7]);
				}
				lastPct=losss;
				lastLoss = myLogLine[6];
                pingArray.push(myLogLine[8]);
				if(lastPing!=0)
					variance += Math.abs(myLogLine[8] - lastPing);
				
				lastPing = myLogLine[8];
            }
            var labelArray = new Array();
            for (i = 1; i < pingArray.length; i++)
                labelArray.push(i);
			var avgPing=0, i=0, sumPing=0,highYAxis=0;
			for(i=0;i<pingArray.length;i++)
				sumPing+=parseInt(pingArray[i]);
			avgPing = Math.floor(sumPing/pingArray.length);
			var bestPing = Math.min.apply(null,pingArray);
			var worstPing = Math.max.apply(null,pingArray);
			var pingIssues = false;
			if (variance > 100) pingIssues = true;

			if (worstPing-bestPing>=35 || avgPing > 100)
				pingIssues = true;
			if (worstPing <= 100)
				highYAxis = 100;
			else highYAxis = worstPing+20;
			var lossPct = 0;
			for (i=0;i<packetLossPctArray.length;i++)
			{
				if(packetLossPctArray[i])
				lossPct+=packetLossPctArray[i];
			}
			lossPct = lossPct / packetLossPctArray.length;
			var worstLoss = Math.max.apply(null,packetLossPctArray);
			var bestLoss = Math.min.apply(null,packetLossPctArray);
			var packetsLostPct = lossPct.toFixed(2)+'%';
			var packetIssues = false;
			if (lossPct > 3 ||
			    worstLoss > 10 ||
				worstLoss - bestLoss >=4)
				packetIssues = true;
			$('.avgping').text(avgPing);
			$('.bestping').text(bestPing);
			$('.worstping').text(worstPing);
			$('.percentpacketloss').text(packetsLostPct);
            var dataLine = {
                labels: labelArray,
                series: [
                    pingArray
                ]
            };
            var optionsLine = {
                low: 0,
				high: highYAxis,
                showPoint: false,
                lineSmooth: false,
                axisY: {
                    showGrid: true,
                    labelInterpolationFnc: function(value) {
                        return value + 'ms';
                    }
                },
                axisX: {
                    showGrid: false,
                    showLabel: false
                }
            };
			var optionsBar = {
                low: 0,
                showPoint: false,
                lineSmooth: false,
                axisY: {
                    showGrid: true,
					showLabel: true,
                    labelInterpolationFnc: function(value) {
                        return value + '%';
                    }
                },
                axisX: {
                    showGrid: false,
                    showLabel: false
                }
            };
			var dataBar={
				labels: labelArray,
				series: [
					packetLossPctArray
				]
			};
			if(packetIssues && pingIssues){
				$('#majorissues').show();
				new Chartist.Line('#majorissues .pingovertime', dataLine, optionsLine);
				new Chartist.Bar('#majorissues .packetlossovertime', dataBar, optionsBar);
			}
			else if (packetIssues){
				$('#somepacketlossissues').show();
				new Chartist.Line('#somepacketlossissues .pingovertime', dataLine, optionsLine);
				new Chartist.Bar('#somepacketlossissues .packetlossovertime', dataBar, optionsBar);
			}
			else if (pingIssues){
				$('#somepingissues').show();
				new Chartist.Line('#somepingissues .pingovertime', dataLine, optionsLine);
				new Chartist.Bar('#somepingissues .packetlossovertime', dataBar, optionsBar);
			}
			else{
				$('#noissues').show();
				new Chartist.Line('#noissues .pingovertime', dataLine, optionsLine);
				new Chartist.Bar('#noissues .packetlossovertime', dataBar, optionsBar);
			}
			$('.pingovertime').css('background', 'white');
			$('.packetlossovertime').css('background', 'white');
			$('.pingovertime').css('height', '190');
			$('.packetlossovertime').css('height', '190');
			$('.pingovertime').css('margin-top', '10px');
			$('.packetlossovertime').css('margin-top', '10px');
  $('html, body').animate({scrollTop: $('hr').first().offset().top}, 2000);
        }
        r.readAsText(f);
    } else { 
        alert("Failed to load file");
    }
}
