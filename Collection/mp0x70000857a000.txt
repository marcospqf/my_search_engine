    <div class="content">
        <div class="ttypography"><p>You are in position (0, 0) of a coordinate plane. You want to determine if you can reach point <span class="tex-span">(<i>x</i>,<i>y</i>)</span>. There are <span class="tex-span"><i>n</i></span> different types of steps that you can make. Step type <span class="tex-span"><i>i</i></span> is represented by two non-negative integers <span class="tex-span"><i>a</i><sub class="lower-index"><i>i</i></sub>,<i>b</i><sub class="lower-index"><i>i</i></sub></span> which means that a step of type <span class="tex-span"><i>i</i></span> moves you <span class="tex-span"><i>a</i><sub class="lower-index"><i>i</i></sub></span> steps to the right and <span class="tex-span"><i>b</i><sub class="lower-index"><i>i</i></sub></span> steps up.</p><p>How to determine if it is possible to reach point <span class="tex-span">(<i>x</i>,<i>y</i>)</span> using only those type of steps? You are allowed to use a type of step multiple times but you can't do half of a step.</p><p>Can this problem be solved with time complexity better than <span class="tex-span"><i>O</i>(<i>x</i>*<i>y</i>*<i>n</i>)</span>?</p></div>
